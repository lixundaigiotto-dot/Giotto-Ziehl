# 巡花问柳大世界：小白版更新手册

这份手册是给以后更新网站用的。
你不需要懂代码，照着做就行。

## 最短理解

以后每次更新网站，你只要记住：

1. 先改内容
2. 本地检查页面
3. 双击 `一键更新网站.bat`

只要脚本上传成功，网站就会自动更新。

---

## 这个项目现在可以搬到别的地方吗

可以。
你可以把整个 `xunhua-world` 文件夹搬到别的位置继续使用。

但要注意：

1. 必须整个文件夹一起移动
2. 不要删掉隐藏的 `.git` 文件夹
3. 如果换了一台电脑，最好先确认这台电脑装了 Git

## 你最常改的地方

### 1. 旅行文字和数据

改这里：

`data/site-data.js`

这里可以改：

- 新增一次旅行
- 修改旅行时间
- 修改停留天数
- 修改照片说明
- 修改视频说明
- 修改四只猫的内容

放这里：

`assets/trips/`

### 3. 猫咪照片

放这里：

`assets/cats/`

### 4. 首页公共图片

放这里：

`assets/shared/`

## 每次更新的正确步骤

### 第 1 步：先改内容

比如你要新增一次旅行：

1. 把照片放进 `assets/trips/`
2. 如果有视频，也放进 `assets/trips/`
3. 打开 `data/site-data.js`
4. 按照原来的格式补一段新的旅行数据

### 第 2 步：本地先看效果

直接双击打开：

`index.html`

重点看这几件事：

- 新照片有没有显示
- 新视频能不能打开
- 国家地图悬浮信息对不对
- 文字有没有写错

如果本地看起来不对，先不要发布。

### 第 3 步：双击一键更新

双击：

`一键更新网站.bat`

它会自动帮你：

- 检查改动
- 保存改动
- 提交改动
- 上传到 GitHub
- 触发网站自动发布

### 第 4 步：等待网站刷新

通常等 1 到 3 分钟，然后打开：

`https://lixundaigiotto-dot.github.io/Giotto-Ziehl/`

刷新网页，就能看到最新内容。

## 常见问题

### 1. 提示没有新的改动

意思是你还没有真正改动文件，或者改了但没保存。

先检查：

- 文件是不是已经保存
- 是不是改错了目录

### 2. 提示没找到 Git

说明这台电脑里没有可用的 Git。

处理方法：

1. 先安装 Git
2. 安装后重新双击 `一键更新网站.bat`

### 3. 提示 push 失败

常见原因：

- 网络有问题
- GitHub 登录状态失效
- 当前电脑没有这个仓库的推送权限

这种情况重新试一次，如果还是不行再处理。

### 4. 网站没有立刻更新

这是正常的。
GitHub Pages 通常要等 1 到 3 分钟。
